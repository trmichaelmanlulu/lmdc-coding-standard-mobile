LMDC Coding Standard (Mobile)

Imports

1. Group your import based on their type using comments

// react modules
import React from 'react';
import { Text, View, } from 'react-native';

// third party libraries
import { Autocomplete, Layout, Icon } from '@ui-kitten/components';

2. The import items order should be like this:

  • React modules
  • Third party libraries
  • Defined Modules
    • Styles
    • Mixins
    • Components
    • Static
// react modules
import React from 'react';
import { Text, View, } from 'react-native';

// third party libraries
import { Autocomplete, Layout, Icon } from '@ui-kitten/components';

// styles
import { styles } from './subscriptions-list-styles-android';

// mixins
import AppHeader from 'lmdc-mobile/src/mixins/app-header';
import AppContent from 'lmdc-mobile/src/mixins/app-content';
import AppRoundedCard from 'lmdc-mobile/src/mixins/app-rounded-card';

// components
import AppFooter from 'lmdc-mobile/src/modules/_shared/components/footer/AppFooter';

// static
import { SCENE } from 'lmdc-mobile/src/static/enums';
import { STRINGS } from 'lmdc-mobile/src/static/strings';

3. Absolute path instead of relative path

Import paths needs to be an absolute path instead of relative path for code readability and maintainability. Using absolute path will lessen the work to update the path when changing the location of your component.

// relative path
import AppFooter from '../../../../modules/_shared/components/footer/AppFooter';

// aboslute path
import AppFooter from 'lmdc-mobile/src/modules/_shared/components/footer/AppFooter';

4. Use the index mixin when importing multiple imxins

// dont
import AppTextInput from 'lmdc-mobile/src/mixins/AppRoundedCard';
import AppDropdown from 'lmdc-mobile/src/mixins/AppTextInput';
import AppAutocomplete from 'lmdc-mobile/src/mixins/AppPrimaryButton';

// do
import { AppTextInput, AppDropdown, AppAutocomplete } from 'lmdc-mobile/src/mixins/index';
