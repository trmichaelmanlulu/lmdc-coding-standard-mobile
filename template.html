LMDC Coding Standard (Mobile)

Template

1. Use these 3 elements to build the base structure of every page screens

  • AppHeader
  • AppContent
  • AppFooter
// dont
<View>
	<View>
		// header
	</View>
	<View>
		<AppContent>
			// content
		</AppContent>
	</View>
	<View>
	    // footer
   </View>
</View>

// do
<View>
    <AppHeader/>
    <AppContent>
        // content
    </AppContent>
    <AppFooter/>
</View>

2. Use the < View > tag to group your elements (View = div)

Think of a view tag as a div element in web development where its main purpose is to group multiple/unrelated elements inside a container.

3. If element attributes is more than 3, separate them in new line

// dont
<AppTextInput value={petName} onChangeText={text => onChange(text, 'petName')} required placeholder="Pet name" />

// do
<AppTextInput
    value={petName}
    placeholder="Pet name"
    required
    onChangeText={text => onChange(text, 'petName')}
/>

4. Element attributes order

Element attributes order should be:

  • Props that has dynamic value
  • Props that has static value
  • Element attribute that has no value
  • Element methods
// dont
<AppTextInput
    placeholder="Pet name"
    accessibilityLabel="PetName"
    value={petName}
    onChangeText={text => onChange(text, 'petName')}
    validations={contentErrors.petName}
    required
    textInputStyle={textInput}
    autoCapitalize="sentences"
/>

// do
<AppTextInput
    value={petName}
    validations={contentErrors.petName}
    textInputStyle={textInput}
    autoCapitalize="sentences"
    placeholder="Pet name"
    accessibilityLabel="PetName"
    required
    onChangeText={text => onChange(text, 'petName')}
/>

5. Always add a propTypes and defaultProps to any component that receives a props value

For example, on your AppTextInput.js mixin, declare your propTypes and defaultProps after the component style.

// propTypes
AppTextInput.propTypes = {
    onChangeText: PropTypes.func.isRequired,
    value: PropTypes.string.isRequired,
    placeholder: PropTypes.string,
    textInputStyle: PropTypes.object,
    hidden: PropTypes.bool,
    accessibilityLabel: PropTypes.string,
}

// defaultProps
AppTextInput.defaultProps = {
    placeholder: 'Enter Here',
    accessibilityLabel: 'TextInput'
}

6. If a template has more than 3 props, use object instead

// dont
export default Template = ({ content, onChange, contentErrors }) => {}

// do
export default Template = (propsObj) => {
   // propsObj.content
   // propsObj.onChange
   // propsObj.contentErrors
}
